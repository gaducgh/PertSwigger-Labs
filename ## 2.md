## 2. SQL injection vulnerability allowing login bypass

### 🎯 Goal
The objective of this challenge was to bypass the authentication process of the application and gain access to the administrator account without knowing the password.

### 🕵️ Methodology (The Approach)
Unlike the previous lab which focused on data retrieval, this task focuses on *Subverting Application Logic*. 

1. *Identifying the Weakness:* I analyzed the login form, which requires a username and password. If the application concatenates these inputs into a query, the backend logic likely follows this structure:
   SELECT * FROM users WHERE username = '...' AND password = '...'

2. *The Logic Bypass:*
   To bypass the password check, I injected a payload into the username field to "neutralize" the password requirement.

### 💀 The Payload
- *Username:* administrator'--
- *Password:* (Anything)

### 🔍 Technical Breakdown (Why it worked?)
* *The Single Quote ('):* This was used to terminate the string for the username field.
* *The Comment (--):* This is the crucial part. By adding the comment operator, the database engine ignores the rest of the query, specifically the AND password = '...' part.
* *Resulting Query:* SELECT * FROM users WHERE username = 'administrator'--' AND password = '...'
  Consequently, the database only verifies if the user administrator exists and ignores the password entirely, allowing successful authentication
